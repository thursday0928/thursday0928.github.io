可能是全网最好用的mac安装MySQL-Python指南

发表于 2021-09-18 更新于 2021-10-09 分类于 工具

在mac下用pip安装MySQL-Python的时候，执行pip install MySQL-python，有时候会遇到如下报错(通常也是最常见的)：

网络上一般提供的方法有：
1.执行export PATH=$PATH:/usr/local/mysql/bin,指定一下路径。
结论: 行不通，殊不知新版的mysql安装路径与这个根本不一样。
2.执行brew install mysql-connector-c
然后再执行pip install mysql-python
结论： 行不通+1
3.（神奇的操作来了）找到自己的mysql安装目录，在其他的某个目录下找到一个叫mysql.h的文件，将其改成my_config.h。。。。。
结论： 行不通+1，会报找不到mysql.h。。。。

以上办法经过证明都是不可行的。终极办法来了：
去网上下一个my_config.h:

curl https://raw.githubusercontent.com/jacobjohn2016/MySQL-Medium/main/my_config.h -o my_config.h

再将该文件放在mysql文件夹下 /usr/local/Cellar/mysql/8.0.21_1/include/mysql
然后再执行pip install MySQL-python，此时我们就发现能安装上了！！！

一个比较全的hexo搭建避坑指南

发表于 2021-04-12 更新于 2021-04-14 分类于 工具

环境

hexo是比较经典且易用的建站工具。可以直接以github为载体，免去申请服务器的步骤。hexo使用markdown解析文章，非常简单好用。
中文文档：https://hexo.io/zh-cn/docs/

首先，要在本地和github建立链接。
这里可以参考这篇来配置：https://segmentfault.com/a/1190000018250408
这篇文章避开了 一个坑。 那就是：
如果按照hexo文档中配置git仓库的方法，也就是repo：https://github.com/username/project 的方式是会失败的。
因为我们一般配置的时候，使用的是ssh公钥，也就是repo: git@github.com:username/username.github.io.git 用这个方式才能配置成功。然后就是注意，branch不是文档中的master，而一般是main。。。这个大家懂得背景。
创建SSH公钥可以参考这篇：https://www.jianshu.com/p/2625e66bb2b2
在使用 ssh-keygen -t rsa -C “github账号的邮箱” 时出现
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/xxxx/.ssh/id_rsa):
这里直接按enter，不要新建一个file了，如果使用新建的file里面的公钥，会链接失败。。会失败。

按照文档中的提示，创建文件夹，开始hexo init。这里没有什么坑。

主题

然后以next这个主题为例，说一下如何配置主题。
next中文文档：https://theme-next.iissnan.com/getting-started.html
或者：https://theme-next.js.org/docs/theme-settings/posts

在执行以下命令的时候，git clone后面还是用ssh链接的好，因为有时候会发现https下载很慢很难成功。

$ cd your-hexo-site
$ git clone https://github.com/iissnan/hexo-theme-next themes/next

上述语句后面的themes/next千万不要丢了。 我因为路径配置错，导致后面的配置死活不生效。

Git链接hexo

git链接hexo需要安装：

npm install hexo-deployer-git --save

不然是无法推送到远端仓库上去的哦。
